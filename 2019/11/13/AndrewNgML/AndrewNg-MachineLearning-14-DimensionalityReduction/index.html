Notes of Andrew Ng’s Machine Learning —— (14) Dimensionality Reduction

Motivation

Data Compression

Dimensionality reduction is another type of unsupervised learning problem.

There are a couple of different reasons why one might want to do dimensionality reduction. One is data compression which not only allows us to compress the data and have it therefore use up less computer memory or disk space, but it will also allow us to speed up our learning algorithms.

What we will do in data compression is reduce data from a high dimensionality to a lower one.

For example:

As we see, to make dimensionality reduce, intuitionally , we projects the high-D on a lower-D.

In the more typical example of dimensionality reduction we might have a thousand dimensional data that we might want to reduce to let’s say a hundred dimensional.

Generally, suppose we apply dimensionality reduction to a dataset of mmm examples {x(1),x(2),⋯ ,x(m)}\{x^{(1)},x^{(2)},\cdots,x^{(m)}\}{x(1),x(2),⋯,x(m)}, where x(i)∈Rnx^{(i)}\in\R^nx(i)∈Rn. As a result of this, we will get out a lower dimensional dataset {z(1),z(2),⋯ ,z(m)}\{z^{(1)},z^{(2)},\cdots,z^{(m)}\}{z(1),z(2),⋯,z(m)} of mmm examples where z(i)∈Rkz^{(i)}\in\R^kz(i)∈Rk for some value of kkk and k≤nk\le nk≤n.

Data Visualization

We can also utilize dimensionality reduction for data visualization.

For a lot of machine learning applications, it really helps us to develop effective learning algorithms, if we can understand our data better. And visualizing the data is a useful in this.

Let’s say we have collected a large data set, which means we have a set of x(i)∈Rnx^{(i)}\in\R^nx(i)∈Rn where nnn is very large, and we can hardly visualize it because, you know, actually we can only draw a plot in or less than 3 dimension. In this case, what we need to do is dimensionality reduction.

For example, we’ve collected a large data set of many statistics about different countries. we may have a huge data set like this:

To visualize this data, what we can do is reduce the dimensionality from 50-D to 2-D, and then we can plot a 2-D plot to understand our data better.

To wrap up, suppose you have a dataset {x(1),x(2),⋯ ,x(m)}\{x^{(1)},x^{(2)},\cdots,x^{(m)}\}{x(1),x(2),⋯,x(m)} where x(i)∈Rnx^{(i)}\in\R^nx(i)∈Rn. In order to visualize it, we apply dimensionality reduction and get {z(1),z(2),⋯ ,z(m)}\{z^{(1)},z^{(2)},\cdots,z^{(m)}\}{z(1),z(2),⋯,z(m)} where z(i)∈Rkz^{(i)}\in\R^kz(i)∈Rk. And we usual make k=2k = 2k=2 or k=3k = 3k=3 (since we can plot 2D or 3D data but don’t have ways to visualize higher dimensional data).

Principal Component Analysis

For the problem of dimensionality reduction, the most commonly used algorithm is something called principal component analysis (PCA).

PCA problem formulation

  • Reduce form 2-dimension to 1-dimension:

    Find a direction (a vector u(1)∈Rnu^{(1)}\in\R^nu(1)∈Rn) onto which to project the data so as to minimize the projection error.

  • Reduce form n-dimension to k-dimension:

    Find kkk vectors u(1),u(2),⋯ ,u(k)u^{(1)},u^{(2)},\cdots,u^{(k)}u(1),u(2),⋯,u(k) onto which to project the data, so as to minimize the projection error.

Notice that PCA is not linear regression! In the linear regression, we minimize the vertical length between the data point and our hypothesis line, while in the PCA, we minimize the distance between the data point and its projected point:

Applying PCA

Reconstruction from Compressed Representation

With PCA, we compressed data from n-dimension to a lower k-dimension. There should be a way to go back from the compressed representation to an approximation of our original high-dimensional data. We call this way reconstruction.

Recall that our PCA is working as z=UreduceTxz= U_{reduce}^Txz=UreduceT​x.

To reconstruction, what we will do is:

xapprox=Ureducezx_{approx}=U_{reduce}z xapprox​=Ureduce​z

It makes z∈Rk→xapprox∈Rnz\in\R^k \to x_{approx}\in\R^nz∈Rk→xapprox​∈Rn. The xapproxx_{approx}xapprox​ will be close to xxx, but we can hardly expect x=xapproxx=x_{approx}x=xapprox​.

Previously, we said that PCA chooses a direction u(1),⋯ ,u(k)u^{(1)},\cdots,u^{(k)}u(1),⋯,u(k) onto which to project the data so as to minimize the (squared) projection error. With our new defined xapproxx_{approx}xapprox​, another way to say the same is that PCA tries to minimize:

1m∑i=1m∣∣x(i)−xapprox(i)∣∣2\frac{1}{m}\sum_{i=1}^{m}||x^{(i)}-x_{approx}^{(i)}||^2 m1​i=1∑m​∣∣x(i)−xapprox(i)​∣∣2

Choosing the kkk (number of principal components)

About PCA, we can see:

  • Average squared projection error: 1m∑i=1m∣∣x(i)−xapprox(i)∣∣2\frac{1}{m}\sum_{i=1}^m||x^{(i)}-x_{approx}^{(i)}||^2m1​∑i=1m​∣∣x(i)−xapprox(i)​∣∣2
  • Total variation in the data: 1m∑i=1m∣∣x(i)∣∣2\frac{1}{m}\sum_{i=1}^{m}||x^{(i)}||^2m1​∑i=1m​∣∣x(i)∣∣2

Typically, choose kkk to be smallest value so that:

1m∑i=1m∣∣x(i)−xapprox(i)∣∣21m∑i=1m∣∣x(i)∣∣2≤0.01\frac{\frac{1}{m}\sum_{i=1}^m||x^{(i)}-x_{approx}^{(i)}||^2}{\frac{1}{m}\sum_{i=1}^{m}||x^{(i)}||^2}\le0.01 m1​∑i=1m​∣∣x(i)∣∣2m1​∑i=1m​∣∣x(i)−xapprox(i)​∣∣2​≤0.01

We call it “99% of variance is retained”.

We can also choose other numbers instead of the usual 0.010.010.01, in practice, 0.01∼0.050.01\sim0.050.01∼0.05 is all capable.

The algorithm utilized for kkk choosing is that:

Try PCA with k=1,⋯ ,nk=1,\cdots,nk=1,⋯,n:

Advice for Applying PCA

Supervised Learning speedup

In supervised learning, we have data like this:

(x(1),y(1)),(x(2),y(2)),⋯ ,(x(m),y(m))(x^{(1)},y^{(1)}),(x^{(2)},y^{(2)}),\cdots,(x^{(m)},y^{(m)}) (x(1),y(1)),(x(2),y(2)),⋯,(x(m),y(m))

We can extract inputs by:

  • Unlabled dataset: x(1),x(2),⋯ ,x(m)∈Rnx^{(1)},x^{(2)},\cdots,x^{(m)}\in \R^nx(1),x(2),⋯,x(m)∈Rn
  • Applying PCA: z(1),z(2),⋯ ,z(m)∈Rkz^{(1)},z^{(2)},\cdots,z^{(m)}\in \R^kz(1),z(2),⋯,z(m)∈Rk, where kkk is lower than nnn

Then we get new training set:

(z(1),y(1)),(z(2),y(2)),⋯ ,(z(m),y(m))(z^{(1)},y^{(1)}),(z^{(2)},y^{(2)}),\cdots,(z^{(m)},y^{(m)}) (z(1),y(1)),(z(2),y(2)),⋯,(z(m),y(m))

Note: Mapping x(i)→z(i)x^{(i)}\to z^{(i)}x(i)→z(i) should be defined by running PCA only on the training set . This mapping can be applied as well to the examples xcv(i)x_{cv}^{(i)}xcv(i)​ and xtest(i)x_{test}^{(i)}xtest(i)​ in the cross validation and test sets.

Application of PCA

  • Compression

    • Reduce memory/disk needed to store data
    • Speed up learning algorithm

    In this case, we should choose kkk by % of variance retained.

  • Visualization

    In this case, k=2k=2k=2 or k=3k=3k=3

Bad use of PCA: To prevent overfitting

Use z(i)z^{(i)}z(i) instead of x(i)x^{(i)}x(i) to reduce the number of features to k<nk<nk<n.

Thus, fewer features, less likely to overfit.

👆 This is a BAD use of PCA.

This might work OK, but isn’t a good way to address overfitting. Use regularization instead:

PCA or not

PCA is sometimes used where it shouldn’t be, here is an example:

When designing a ML system, we wrote down this steps:

  1. Get training set {(x(1),y(1)),⋯ ,(x(m),y(m))}\{(x^{(1)},y^{(1)}),\cdots,(x^{(m)},y^{(m)})\}{(x(1),y(1)),⋯,(x(m),y(m))}
  2. Run PCA to reduce x(i)x^{(i)}x(i) in dimension to get ziz^{i}zi
  3. Train logistic regression on {(z(1),y(1)),⋯ ,(z(m),y(m))}\{(z^{(1)},y^{(1)}),\cdots,(z^{(m)},y^{(m)})\}{(z(1),y(1)),⋯,(z(m),y(m))}
  4. Test on test set: Map xtest(i)x_{test}^{(i)}xtest(i)​ to ztest(i)z_{test}^{(i)}ztest(i)​. Run hθ(z)h_\theta(z)hθ​(z) on {(ztest(1),y(1)),⋯ ,(ztest(m),y(m))}\{(z_{test}^{(1)},y^{(1)}),\cdots,(z_{test}^{(m)},y^{(m)})\}{(ztest(1)​,y(1)),⋯,(ztest(m)​,y(m))}

But before doing this, we should think: How about the whole thing without using PCA?

Before implementing PCA, first try running whatever you want to do with the original/raw data x(i)x^{(i)}x(i). Only if that doesn’t do what you want, then implement PCA and consider using z(i)z^{(i)}z(i).
