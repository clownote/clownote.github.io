Density Estimation

Problem Motivation

Anomaly Detection is a type of machine learning problem.

Imagine that you’re a manufacturer of aircraft engines, and let’s say that as your aircraft engines roll off the assembly line, you’re doing quality assurance testing, and as part of that testing you measure features of your aircraft engine, like the heat generated, the vibrations and so on. So you now have a data set of x1x_1x1​ through xmx_mxm​, if you have manufactured mmm aircraft engines and plot your data. Then, given a new engine xtestx_{test}xtest​, you want to know is xtestx_{test}xtest​ anomalous? This problem is called Anomaly Detection.

Here is want we are going to do:

Dataset: {x(1),x(2),⋯ ,x(m)}\{x^{(1)},x^{(2)},\cdots,x^{(m)}\}{x(1),x(2),⋯,x(m)}

Is xtestx_{test}xtest​ anomalous?

To solve this problem, we will train a model for p(x)p(x)p(x) (i.e. a model for the probability of xxx, where xxx are these features of, say, aircraft engines). We’re then going to say that for the new aircraft engine, if p(xtest)p(x_{test})p(xtest​) is less than some ϵ\epsilonϵ then we flag this as an anomaly:

p(xtest)<ϵ⇒anomalyp(xtest)≥ϵ⇒OK\begin{array}{l} p(x_{test})<\epsilon \Rightarrow \textrm{anomaly}\\ p(x_{test})\ge\epsilon \Rightarrow \textrm{OK} \end{array} p(xtest​)<ϵ⇒anomalyp(xtest​)≥ϵ⇒OK​

This problem of estimating this distribution p(x)p(x)p(x) are sometimes called the problem of density estimation.

Gaussian Distribution

Gaussian distribution which is also called the normal distribution.

If xxx is a distributed Gaussian with mean μ\muμ, variance σ2\sigma^2σ2, we will write it as:

x∼N(μ,σ2)x \sim \mathcal{N}(\mu,\sigma^2) x∼N(μ,σ2)

In this case, the probability of xxx is:

p(x)=12πσexp⁡(−(x−μ)22σ2)p(x) = \frac{1}{\sqrt{2\pi}\sigma}\exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right) p(x)=2π​σ1​exp(−2σ2(x−μ)2​)

μ=1m∑i=1mx(i)\mu=\frac{1}{m}\sum_{i=1}^{m}x^{(i)} μ=m1​i=1∑m​x(i)

σ2=1m∑i=1m(x(i)−μ)2\sigma^2=\frac{1}{m}\sum_{i=1}^{m}\left(x^{(i)}-\mu\right)^2 σ2=m1​i=1∑m​(x(i)−μ)2

Building an Anomaly Detection System

Developing and Evaluating an Anomaly Detection System

The importance of real-number evalua-on

When developing a learning algorithm (choosing features, etc.), making decisions is much easier if we have a way of evalua-ng our learning algorithm.

Assume we have some labeled data, of anomalous and non- anomalous examples. (y=0y=0y=0 if normal, y=1y=1y=1 if anomalous).

  • Training set: x(1),x(2),⋯ ,x(m)x^{(1)},x^{(2)},\cdots,x^{(m)}x(1),x(2),⋯,x(m) (assume normal examples/not anomalous)

  • Cross validation set: (xcv(1),ycv(1)),⋯ ,(xcv(mcv),ycv(mcv))\left(x_{cv}^{(1)},y_{cv}^{(1)}\right),\cdots,\left(x_{cv}^{(m_{cv})},y_{cv}^{(m_{cv})}\right)(xcv(1)​,ycv(1)​),⋯,(xcv(mcv​)​,ycv(mcv​)​)

  • Test set: (xtest(1),ytest(1)),⋯ ,(xtest(mtest),ytest(mtest))\left(x_{test}^{(1)},y_{test}^{(1)}\right),\cdots,\left(x_{test}^{(m_{test})},y_{test}^{(m_{test})}\right)(xtest(1)​,ytest(1)​),⋯,(xtest(mtest​)​,ytest(mtest​)​)

Aircraft engines motivating example

Say, what we have is:

  • 10000 good (normal) engines (y=0y=0y=0)
  • 20 flawed engines (anomalous) (y=1y=1y=1)

Than we are going to choose:

  • Training set: 6000 good engines
  • CV: 2000 good engines (y=0y=0y=0), 10 anomalous (y=1y=1y=1)
  • Test: 2000 good engines (y=0y=0y=0), 10 anomalous (y=1y=1y=1)

Algorithm evaluation

Fit model p(x)p(x)p(x) on training set {x(1),⋯ ,x(m)}\{x^{(1)},\cdots,x^{(m)}\}{x(1),⋯,x(m)}

On a cross validation/test example xxx, predict:

y={1if p(x)<ϵ(anomaly)0if p(x)≥ϵ(normal)y = \left\{\begin{array}{ll} 1 & \textrm{if } p(x) < \epsilon \quad \textrm{(anomaly)}\\ 0 & \textrm{if } p(x) \ge \epsilon \quad \textrm{(normal)}\\ \end{array}\right. y={10​if p(x)<ϵ(anomaly)if p(x)≥ϵ(normal)​

Possible evaluation matrices:

  • True positive, false positive, false negative, true negative
  • Precision/Recall
  • F1F_1F1​-score

Can also use cross valida-on set to choose parameter ϵ\epsilonϵ.

Anomaly Detection vs. Supervised Learning

There are some more examples:

  • (Anomaly Detection) You run a power utility (supplying electricity to customers) and want to monitor your electric plants to see if any one of them might be behaving strangely.

  • (Supervised Learning) You run a power utility and want to predict tomorrow’s expected demand for electricity (so that you can plan to ramp up an appropriate amount of generation capacity).

  • (Anomaly Detection) A computer vision / security application, where you examine video images to see if anyone in your company’s parking lot is acting in an unusual way.

  • (Supervised Learning) A computer vision application, where you examine an image of a person entering your retail store to determine if the person is male or female.

Error analysis for anomaly detection

What we want is:

  • p(x)p(x)p(x) large for normal examples xxx.
  • p(x)p(x)p(x) small for anomalous examples xxx.

And the most common problem is:

p(x)p(x)p(x) is comparable (say, both large) for normal and anomalous examples.

Suppose our anomaly detection algorithm is performing poorly and outputs a large value of p(x) for many normal examples and for many anomalous examples in the cross validation dataset, what is most likely to help is to try coming up with more features to distinguish between the normal and the anomalous examples.

Multivariate Gaussian distribution

Multivariate Gaussian Distribution

What we still want to do is:

  • Given x∈Rnx\in\R^nx∈Rn.
  • Don’t model p(x1),p(x2),⋯p(x_1),p(x_2),\cdotsp(x1​),p(x2​),⋯ separately.
  • Model p(x)p(x)p(x) all in one go.

Multivariate gaussian is able to make it:

p(x;μ,Σ)=exp⁡(−12(x−μ)TΣ−1(x−μ))(2π)n∣Σ∣p(x;\mu,\Sigma)=\frac {\exp\left(-\frac{1}{2}(x-\mu)^T\Sigma^{-1}(x-\mu)\right)} {\sqrt{(2\pi)^{n}|\Sigma|}} p(x;μ,Σ)=(2π)n∣Σ∣​exp(−21​(x−μ)TΣ−1(x−μ))​

P.s. $|\Sigma| = \det\Sigma $ is the determinant of Σ\SigmaΣ .

Where we need parameters:

  • μ∈Rn\mu\in\R^nμ∈Rn
  • Σ∈Rn×n\Sigma\in\R^{n\times n}Σ∈Rn×n (covariance matrix, Sigma = 1/m * X' * X;)

Given training set {x(1),⋯ ,x(m)}\{x^{(1)},\cdots,x^{(m)}\}{x(1),⋯,x(m)}, we can fit the parameters:

μ=1m∑i=1mx(i)Σ=1m∑i=1m(x(i)−μ)(x(i)−μ)T\mu=\frac{1}{m}\sum_{i=1}^mx^{(i)} \qquad \Sigma=\frac{1}{m}\sum_{i=1}^m\left(x^{(i)}-\mu\right)\left(x^{(i)}-\mu\right)^T μ=m1​i=1∑m​x(i)Σ=m1​i=1∑m​(x(i)−μ)(x(i)−μ)T

Relationship to original model

Original model: p(x)=p(x1;μ1,σ12)×p(x2;μ2,σ22)×⋯×p(xn;μn,σn2)p(x)=p(x_1;\mu_1,\sigma_1^2)\times p(x_2;\mu_2,\sigma_2^2) \times \cdots \times p(x_n;\mu_n,\sigma_n^2)p(x)=p(x1​;μ1​,σ12​)×p(x2​;μ2​,σ22​)×⋯×p(xn​;μn​,σn2​)

As we can see, the contours of the Original model are always axis aligned.

This model actually corresponds to a special case of a multivariate Gaussian distribution p(x)=exp⁡(−12(x−μ)TΣ−1(x−μ))(2π)n∣Σ∣p(x)=\frac{\exp\left(-\frac{1}{2}(x-\mu)^T\Sigma^{-1}(x-\mu)\right)} {\sqrt{(2\pi)^{n}|\Sigma|}}p(x)=(2π)n∣Σ∣​exp(−21​(x−μ)TΣ−1(x−μ))​ where:

Σ=[σ12σ12⋱σn2]\Sigma=\left[\begin{array}{cccc} \sigma_1^2 \\ & \sigma_1^2 \\ & & \ddots \\ & & & \sigma_n^2 \end{array}\right] Σ=⎣⎢⎢⎡​σ12​​σ12​​⋱​σn2​​⎦⎥⎥⎤​

Original model vs. Multivariate gaussian

Original model:

  • p(x)=p(x1;μ1,σ12)×⋯×p(xn;μn,σn2)p(x)=p(x_1;\mu_1,\sigma_1^2)\times \cdots \times p(x_n;\mu_n,\sigma_n^2)p(x)=p(x1​;μ1​,σ12​)×⋯×p(xn​;μn​,σn2​)
  • Manually create features to capture anomalies where x1,x2x_1,x_2x1​,x2​ take unusual combinations of values. (e.g. x3=x1x2x_3=\frac{x_1}{x_2}x3​=x2​x1​​)
  • Computationally cheaper (alternatively, scales better to large nnn)
  • OK even if mmm (training set size) is small

Multivariate gaussian:

  • p(x)=exp⁡(−12(x−μ)TΣ−1(x−μ))(2π)n∣Σ∣p(x)=\frac{\exp\left(-\frac{1}{2}(x-\mu)^T\Sigma^{-1}(x-\mu)\right)} {\sqrt{(2\pi)^{n}|\Sigma|}}p(x)=(2π)n∣Σ∣​exp(−21​(x−μ)TΣ−1(x−μ))​
  • Automatically captures correlations between features (no need to create a x3=x1x2x_3=\frac{x_1}{x_2}x3​=x2​x1​​)
  • Computationally more expensive
  • Must have m>nm>nm>n & all features are not redundant (need to promise that there are no features that are linearly dependent) or else Σ\SigmaΣ is non-invertible.
