Notes of Andrew Ng’s Machine Learning —— (3) Multivariate Linear Regression

Multiple Features

Linear regression with multiple variables is alse known as “multivariate linear regression”.

Notations

We now introduce notation for equations where we can have any number of input variables (Multiple Features, i.e. Multivariate):

  • mmm: the number of training examples.
  • nnn: the number of features.
  • x(i)x^{(i)}x(i): the input (features) of the ithi^{th}ith training example, a n-dimensional vector.
  • xj(i)x^{(i)}_jxj(i)​: the value of feature jjj in the ithi^{th}ith training example.

Hypothesis

The multivariable form of the hypothesis function accommodating these multiple features is as follows:

hθ(x)=θ0+θ1x1+θ2x2+θ3x3+...+θnxnh_\theta(x)=\theta_0+\theta_1x_1+\theta_2x_2+\theta_3x_3+ ... + \theta_nx_n hθ​(x)=θ0​+θ1​x1​+θ2​x2​+θ3​x3​+...+θn​xn​

In order to develop intuition about this function, we can think about θ0\theta_0θ0​ as the basic price of a house, θ1\theta_1θ1​ as the price per square meter, θ2\theta_2θ2​ as the price per floor, etc. x1x_1x1​ will be the number of square meters in the house, x2x_2x2​ the number of floors, etc.

Using the definition of matrix muyltiplication, our multivariable hypothesis function can be concisely represented as:

hθ(x)=[θ0θ1…θn][x0x1⋮xn]=θTxh_\theta(x)= \left[\begin{array}{c}\theta_0 & \theta_1 & \ldots & \theta_n\end{array}\right] \left[\begin{array}{c}x_0 \\ x_1 \\ \vdots \\ x_n \end{array}\right] = \theta^Tx hθ​(x)=[θ0​​θ1​​…​θn​​]⎣⎢⎢⎢⎡​x0​x1​⋮xn​​⎦⎥⎥⎥⎤​=θTx

This is a vectorization of our hypothesis function for one training example.

Remark: Note that for convenience reasons, we assume x0(i)=1for(i∈1,...,m)x^{(i)}_0=1 \quad \textrm{for(}i \in 1, ...,m \textrm{)}x0(i)​=1for(i∈1,...,m). This allows us to do matrix operations with θ\thetaθ and xxx. Hence making two vector θ\thetaθ and x(i)x^{(i)}x(i) match each other element-wise (that is, have the same number of elements: n+1).

Feature Scaling

We can speed up gradient descent by having each of our input values in roughly the same range. This is because θ\thetaθ will descend quickly on small ranges and slowly on large ranges, and so will oscillate inefficiently down to the optimum when the variables are very uneven.

The way to prevent this is to modify the ranges of our input variables so that they are all roughly the same. Ideally:

−1≤xi≤1or−0.5≤xi≤0.5\begin{array}{c} -1 \le x_{i} \le 1\\ \textrm{or}\\ -0.5 \le x_{i} \le 0.5 \end{array} −1≤xi​≤1or−0.5≤xi​≤0.5​

These aren’t exact requirements; we are only trying to speed things up. The goal is to get all input variables into roughly one of these ranges, give or take a few.

In practice, we offen think it’s ok for variables in range [−3,−13)∪(+13,+3][-3,-\frac{1}{3}) \cup (+\frac{1}{3}, +3][−3,−31​)∪(+31​,+3].

Two techniques to help with this are feature scaling and mean normalization.

Feature scaling

Feature scaling involves dividing the input values by the range (i.e. the maximum value minus the minimum value) of input variable, resulting in a new range of just 1.

Range:si=max(xi)−min(xi)Scaling:xi:=xisi\begin{array}{rl} \textrm{Range:} & s_i = max(x_i)-min(x_i)\\ \textrm{Scaling:} & x_i:=\frac{x_i}{s_i} \end{array} Range:Scaling:​si​=max(xi​)−min(xi​)xi​:=si​xi​​​

Mean normalization

Mean normalization involves subtracting the average value for an input variable from the values for that input variable resulting in a new average value for the input variable of just zero.

Average:μi=sum(xi)mMean normalizing:xi:=xi−μi\begin{array}{rl} \textrm{Average:} & \mu_i = \frac{sum(x_i)}{m}\\ \textrm{Mean normalizing:} & x_i:=x_i-\mu_i \end{array} Average:Mean normalizing:​μi​=msum(xi​)​xi​:=xi​−μi​​

Implement

We always implement both of these techniques via adjusting our input values as shown in this formula:

xi:=xi−μisix_i:=\frac{x_i-\mu_i}{s_i} xi​:=si​xi​−μi​​

  • μi\mu_iμi​ is the average of all the values for feature (i)
  • sis_isi​ is the range of values (max - min), or sis_isi​ could also be the standard deviation.

For example, if xix_ixi​ represents housing prices with a range of 100 to 2000 and a mean value of 1000, then, xi:=price−10001900x_i := \dfrac{price-1000}{1900}xi​:=1900price−1000​.

Learning Rate

Debugging gradient descent

Make a plot with number of iterations on the x-axis. Now plot the cost functiong J(θ)J(\theta)J(θ) over the number of iterations of gradient descent. If J(θ)J(\theta)J(θ) ever increases, then you probably need to decrease learning rate α\alphaα.

Automatic convergence test

Declare convergence if J(θ)J(\theta)J(θ) decreases by less than ϵ\epsilonϵ, where ϵ\epsilonϵ is some small value such as 10−310^{-3}10−3. However in practice it’s difficult to choose this threshold value.

Making sure gradient descent is working correctly

It has been proven that if learning rate α is sufficiently small, then J(θ) will decrease on every iteration.

To summarize:

If α\alphaα is too small: slow convergence.

If α\alphaα is too large: ￼may not decrease on every iteration and thus may not converge.

Features and Polynomial Regression

Combine Features

We can improve our features and the form of our hypothesis function in a couple different ways.

For example, we can combine multiple features into one, Such as combining x1x_1x1​ and x2x_2x2​ into a new feature x3x_3x3​ by taking x1⋅x2x_1 \cdot x_2x1​⋅x2​.

Polynomial Regression

To fit the data well, our hypothesis function may need to be non-linear. So we can change the behavior or curve of our hypothesis function by making it a quadratic, cubic or square root function (or any other form).

For example, if our hypothesis function is hθ(x)=θ0+θ1x1h_\theta(x)=\theta_0+\theta_1x_1hθ​(x)=θ0​+θ1​x1​ then we can create additional features based on x1x_1x1​, to get the quadratic function hθ(x)=θ0+θ1x1+θ2x12h_\theta(x)=\theta_0+\theta_1x_1+\theta_2x_1^2hθ​(x)=θ0​+θ1​x1​+θ2​x12​ or the cubic function hθ(x)=θ0+θ1x1+θ2x12+θ3x13h_\theta(x)=\theta_0+\theta_1x_1+\theta_2x_1^2+\theta_3x_1^3hθ​(x)=θ0​+θ1​x1​+θ2​x12​+θ3​x13​, to make it a square root function, we could do: hθ(x)=θ0+θ1x1+θ2x1h_\theta(x) = \theta_0 + \theta_1 x_1 + \theta_2 \sqrt{x_1}hθ​(x)=θ0​+θ1​x1​+θ2​x1​​.

In the cubic version, we can create new features x2x_2x2​, x3x_3x3​ where x2=x12x_2=x_1^2x2​=x12​ and x3=x13x_3=x_1^3x3​=x13​, then we can get a set of thetas via gradient descent for multiple variables.
